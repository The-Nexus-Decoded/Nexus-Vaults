## Lessons Learned

- **Systemd + venv = explicit wrapper:** Never assume systemd's `ExecStart` will respect a virtual environment. Use a wrapper script that activates the venv and sets `PATH`/`PYTHONPATH` explicitly. Document this pattern in deployment guides.
- **Jupiter v6 transaction format changed:** Versioned transactions with compressed accounts require ALTs. The API response includes `addressLookupTableAddresses` that must be passed to `sendTransaction` (or `sendRawTransaction`). Discarding this field causes silent failures that look like "invalid transaction" errors.
- **Dependency installation location matters:** `httpx` being present in the venv is insufficient if the service doesn't use that venv. Always verify the runtime environment, not just the venv state.
